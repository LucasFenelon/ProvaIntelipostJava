1 - Imagine que hoje temos um sistema de login e perfis de usuários, esse sistema conta com mais de 10 milhões de usuários. Hoje a tela inicial do sistema se encontra muito lenta. Nessa tela é feita uma consulta no banco de dados para pegar as informações do usuário e exibi-las de forma personalizada. Quando há um pico de logins simultâneos, este também fica lento. Na sua visão que tipo de melhoria poderia ser feita?

Pensando na necessidade de inúmeros acessos simultâneos, em uma volumetria da ordem dos milhões, podemos colocar parte dos dados de acesso em um banco de dados noSql, como dados de autenticação, por exemplo, podemos copiar a parte necessária, como login e senha para um banco chave/valor e permitir um acesso muito mais rápido, num segundo momento, conforme for necessário buscar os demais dados, quando um usuário acessar a tela de alteração de cadastros, podemos buscar os mesmo dados em uma base de dados relacional. No momento do cadastro de um novo usuário temos de considerar a persistencia nas duas bases de dados.
